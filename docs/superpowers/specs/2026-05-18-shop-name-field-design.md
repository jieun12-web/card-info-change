# 상호명 필드 추가 설계

날짜: 2026-05-18
대상: 카드사 정보변경 신청 페이지 (`index.html`, `apps-script.gs`)

## 목적

카드사 정보변경 신청 폼에서 신청자의 **상호명**을 함께 수집한다.

## 요구사항

- 위치: 사업자번호 입력 칸 바로 아래, 휴대폰 번호 칸 위 (첫 번째 카드 내부)
- 입력 형식: 자유 텍스트. 길이 제한 없음. 한글·영문·숫자 모두 허용 (입력 필터 없음)
- 필수 입력: 공백 trim 후 길이가 0이면 제출 차단 + 오류 메시지 표시
- 구글시트 C열에 저장 (열은 사용자가 이미 추가해 둠)

## 변경 내용

### 1. `index.html`

- `#f-biz`와 `#f-phone` 사이에 `#f-name` 필드 추가
  - `<input id="name" name="name" type="text">`, `maxlength` 없음
  - 라벨 "상호명" + 필수 표시 `*`
  - 오류 메시지: "상호명을 입력해주세요."
- 검증 로직 `validate()`에 상호명 필수 체크 추가 (`name.trim().length > 0`)
- 제출 `payload`에 `name: nameEl.value.trim()` 추가
- 숫자 전용 필터(`onlyDigits`)는 상호명에 **적용하지 않음**

### 2. `apps-script.gs`

시트 헤더(사용자가 구성한 최종 순서):

| A | B | C | D | E | F | G |
|---|---|---|---|---|---|---|
| 접수시각 | 사업자번호 | 상호명 | 휴대폰 | 변경항목 | 기타사항 | 처리완료 |

- `appendRow` 배열을 `[접수시각, "'"+biz, name, "'"+phone, items.join(', '), etc]` 순으로 변경
  - 상호명은 텍스트이므로 `'` 강제 텍스트 접두어 불필요
- 처리완료 체크박스 삽입 위치를 6열(F) → 7열(G)로 변경
- 상단 헤더 주석을 새 열 구성으로 갱신

## 테스트 (수동)

- 상호명 미입력 시 제출 차단 + 오류 메시지 노출 확인
- 한글/영문/숫자 혼합 상호명 입력 후 제출 → 시트 C열에 정확히 저장, 나머지 열(휴대폰·변경항목·기타사항·처리완료) 밀림 없음 확인
- 처리완료 체크박스가 G열에 생성되는지 확인

## 범위 밖

- 상호명 형식·중복 검증 (단순 빈 값 체크만)
- 기존 시트 데이터 마이그레이션 (신규 접수분부터 적용)
