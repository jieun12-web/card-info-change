# [내죠여왕] 자동 적재 설계

날짜: 2026-05-18
대상: `apps-script.gs` (기존 Google Apps Script에 `onEdit` 추가)

## 목적

소스 시트 `시트1`에서 G열(처리완료) 체크박스를 켜면, 같은 스프레드시트의 `내죠여왕` 시트에 가공된 1행을 자동으로 추가한다. 행당 1회만 적재한다.

## 트리거

container-bound 단순 `onEdit(e)` 트리거. 다음 조건을 **모두** 만족할 때만 동작:

- `e.range.getSheet().getName() === '시트1'`
- 편집된 열 == 7 (G), 편집된 단일 셀
- 새 값이 체크됨(`e.value === 'TRUE'` 또는 `e.range.getValue() === true`)
- 행 번호 >= 2 (데이터 행)
- 소스 `시트1` 같은 행의 **H열(8)이 비어 있음** → 이미 처리된 행이면 즉시 종료(행당 1회 보장)

조건 불충족 시 아무 동작 없이 `return`.

## 소스(시트1) 열 참조

| 열 | 내용 | 저장 형태 |
|---|---|---|
| B (2) | 사업자번호 | `'1234567890` (텍스트 강제 접두어, 값은 숫자 문자열) |
| C (3) | 상호명 | 텍스트 |
| D (4) | 휴대폰 | `'01012345678` (텍스트, 앞 0 유지) |
| G (7) | 처리완료 체크박스 | boolean |
| H (8) | 처리 표식(본 기능이 기록) | 비어있음 → 미처리 |

## [내죠여왕]에 추가할 행

`내죠여왕` 시트의 **마지막 내용 행 다음 행**에 다음 6개 열을 기록(`getLastRow()+1`, 헤더 유무와 무관):

| 내죠여왕 열 | 값 | 산출 방법 |
|---|---|---|
| A | 상호명 | 시트1 C열 값 |
| B | 휴대폰 | 시트1 D열 값을 `"'" + 값`으로 텍스트 강제 (앞 0 유지) |
| C | MOBIL 사번 | `"MOBIL" + MMDD + "-" + zeroPad4(순번)` |
| D | `정보변경` | 고정 문자열 |
| E | 금액 (숫자) | 시트1 B열에서 숫자만 추출 → 4·5번째 자리 두 자리 정수가 81~87이면 `55000`, 그 외 `33000` |
| F | 만료일 8자리 | `(스크립트 실행일 + 29일)`을 `yyyyMMdd` 문자열로 (텍스트 강제) |

### C열 순번 산출

- `MMDD` = 스크립트 실행일의 월(2자리)+일(2자리), 타임존 `Asia/Seoul`
- 접두어 `prefix = "MOBIL" + MMDD + "-"`
- `내죠여왕` C열 전체 값 중 `prefix`로 시작하는 셀 개수 `n`
- 새 순번 = `n + 1`, 4자리 zero-pad → 예 첫 건 `MOBIL0518-0001`

### E열 금액 산출

```
digits = 시트1 B열 값에서 [^0-9] 전부 제거
pair  = parseInt(digits.substring(3,5), 10)   // 4·5번째 자리(0-index 3,4)
금액  = (pair >= 81 && pair <= 87) ? 55000 : 33000
```
(81,82,83,84,85,86,87 = 법인 코드. 검증 예: `1234567890`→`45`→33000, `1238167890`→`81`→55000, `5558767890`→`87`→55000)

### F열 만료일

`Utilities.formatDate(new Date(now + 29일), 'Asia/Seoul', 'yyyyMMdd')`, 텍스트로 기록.

## 처리 표식

행 추가 성공 후, 소스 `시트1`의 해당 행 **H열(8)**에 생성된 MOBIL 사번 문자열을 기록. 이후 같은 행에서 체크박스를 껐다 다시 켜도 H열이 채워져 있어 스킵.

## 동시성/순서

저용량 수동 작업이므로 락 없이 진행. 만약 안정성 강화가 필요하면 `LockService` 추가는 범위 밖(YAGNI).

## 기존 코드 영향

`doPost`(폼 → 시트1 저장), `doGet`은 변경 없음. `onEdit` 함수만 신규 추가. 단순 트리거라 별도 권한 승인 불필요.

## 테스트 (수동)

1. 시트1 데이터 행에서 G열 체크 → 내죠여왕 마지막 행 다음에 A~F가 규칙대로 채워짐, 시트1 H열에 MOBIL사번 기록
2. 같은 행 체크 해제 후 재체크 → 내죠여왕에 중복 행 추가되지 않음
3. 사업자번호 4·5자리 81~87 / 그 외 각각 → E열 55000 / 33000 확인
4. 같은 날 2건 이상 처리 → C열 순번 0001, 0002 … 증가
5. 휴대폰 앞 0 유지(B열 텍스트) 확인
6. F열 = 처리일 + 29일, yyyyMMdd 8자리 확인

## 범위 밖

- 행 삭제/수정 시 내죠여왕 동기화
- LockService 동시성 처리
- 내죠여왕 헤더 자동 생성
- 시트1 외 다른 시트에서의 트리거
